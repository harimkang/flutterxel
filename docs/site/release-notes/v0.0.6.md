# Release v0.0.6

Release date: 2026-03-04

## Published Packages

- `flutterxel` 0.0.6: https://pub.dev/packages/flutterxel
- `flutterxel_tools` 0.0.6: https://pub.dev/packages/flutterxel_tools

Git tag:

- `v0.0.6`

## Highlights

- Added backend discriminator ABI and stricter ABI parity checks across native/plugin/Dart surfaces.
- Added runtime backend introspection:
  - `BackendMode`
  - `Flutterxel.backendMode`
  - `Flutterxel.supportsNativeBltSourceSelection`
- Added deterministic fallback test forcing with `FLUTTERXEL_FORCE_BACKEND` and `FLUTTERXEL_LIBRARY_OVERRIDE`.
- Fixed C fallback source selection so:
  - `blt` honors `img`
  - `bltm` honors `tm`
- Added opt-in alpha-aware image import policy and documented transparent-index + `colkey` recipes.
- Added Agent Map MVP example pipeline (`character_manifest`, state machine, renderer/controller, JSONL activity adapter) with tests.

## Upgrade Notes

### For package users

- Existing defaults remain Pyxel-compatible; alpha handling changes are opt-in via new `Image.load`/`Image.fromImage` options.
- Backend-specific branching can now use `Flutterxel.backendMode` directly.
- `c_fallback` rendering source-selection behavior now matches image/tilemap id expectations.

### For maintainers

- Release validation path remains tag-triggered CI dry-run (`publish_dry_run.yml`).
- pub.dev publishing is executed manually per package after CI dry-run succeeds.

# Release v0.0.11

Release date: 2026-03-06

## Published Packages

- `flutterxel` 0.0.11: https://pub.dev/packages/flutterxel
- `flutterxel_tools` 0.0.11: https://pub.dev/packages/flutterxel_tools

Git tag:

- `v0.0.11`

## Highlights

- Added runtime palette count expansion across native and Dart runtime paths (`16/64/256`).
- Added runtime ABI symbols for palette count configuration:
  - `flutterxel_core_set_num_colors`
  - `flutterxel_core_num_colors`
- Added `init(..., num_colors / numColors)` and runtime getters (`num_colors` / `numColors`).
- Updated `pal`, `load_pal`, and `save_pal` behavior to respect runtime palette count.
- Updated `FlutterxelView` default palette path to follow runtime palette size when `palette` is omitted.
- Refreshed bundled Android/iOS native artifacts for the expanded ABI.

## Upgrade Notes

### For package users

- Existing code remains backward compatible when `num_colors` is omitted (defaults to `16`).
- To opt in:
  - `flutterxel.init(..., num_colors: 64)`
  - `flutterxel.init(..., numColors: 256)`
- If a native backend reports missing symbol errors for `set/get_num_colors`, use artifacts built from this release.

### For maintainers

- Version metadata is aligned at `0.0.11` in:
  - `packages/flutterxel/pubspec.yaml`
  - `packages/flutterxel_tools/pubspec.yaml`
  - `packages/flutterxel/ios/flutterxel.podspec`
  - `packages/flutterxel/src/CMakeLists.txt`
- Native artifact layout remains:
  - Android: `packages/flutterxel/native/android/jniLibs/...`
  - iOS: `packages/flutterxel/ios/Frameworks/FlutterxelCore.xcframework`

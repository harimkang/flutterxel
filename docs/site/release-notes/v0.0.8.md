# Release v0.0.8

Release date: 2026-03-04

## Published Packages

- `flutterxel` 0.0.8: https://pub.dev/packages/flutterxel
- `flutterxel_tools` 0.0.8: https://pub.dev/packages/flutterxel_tools

Git tag:

- `v0.0.8`

## Highlights

- Fixed iOS backend symbol shadowing by removing fallback `flutterxel_core_*` exports from plugin C sources.
- Repackaged iOS native core bundle to `ios/Frameworks/FlutterxelCore.xcframework`.
- Standardized xcframework slice library names to `libflutterxel_core.a` for CocoaPods compatibility.
- Updated iOS podspec to vendor the xcframework directly and force-load static slices.
- Updated Dart loader behavior on iOS to try explicit library candidates before `DynamicLibrary.process()`.
- Updated CI native artifact workflow and overlay paths for the new iOS framework location.

## Upgrade Notes

### For package users

- Public runtime API is unchanged.
- iOS consumers should use the `0.0.8` package so Rust core symbols resolve from the vendored xcframework path.

### For maintainers

- Native artifact layout now spans:
  - Android: `packages/flutterxel/native/android/jniLibs/...`
  - iOS: `packages/flutterxel/ios/Frameworks/FlutterxelCore.xcframework`
- Release validation remains: tag-triggered CI dry-run plus manual pub.dev publish.

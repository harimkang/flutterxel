# Release v0.0.5

Release date: 2026-03-04

## Published Packages

- `flutterxel` 0.0.5: https://pub.dev/packages/flutterxel
- `flutterxel_tools` 0.0.5: https://pub.dev/packages/flutterxel_tools

Git tag:

- `v0.0.5`

## Highlights

- Fixed native-render path mismatch where resource image mutations (`images[n].pset/cls/load/set`) were not reflected by global `blt(...)`.
- Added explicit image resource mutation/read ABI:
  - `flutterxel_core_image_pset`
  - `flutterxel_core_image_pget`
  - `flutterxel_core_image_cls`
- Added regression tests for native resource image sync behavior in `flutterxel_api_surface_test.dart`.
- Added CI workflow for actual pub.dev publishing on release tags:
  - `.github/workflows/publish_pub_dev.yml`

## Upgrade Notes

### For package users

- Global `blt(...)` source contract is unchanged: use image resource ids (`int`) or resource-backed `Image` objects.
- Detached image objects (`Image(...)`, `Image.fromImage(...)`) should still be drawn with `image.blt(...)`.
- Built-in text rendering remains ASCII-only (`32..127`) and skips unsupported code points.

### For maintainers

- Release publish pipeline now has separate dry-run and publish workflows:
  - `publish_dry_run.yml`
  - `publish_pub_dev.yml`
- Set repository secret `PUB_DEV_PUBLISH_TOKEN` for CI-based publish.

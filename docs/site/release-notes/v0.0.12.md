# Release v0.0.12

Release date: 2026-03-06

## Published Packages

- `flutterxel` 0.0.12: https://pub.dev/packages/flutterxel
- `flutterxel_tools` 0.0.12: https://pub.dev/packages/flutterxel_tools

Git tag:

- `v0.0.12`

## Highlights

- Added runtime `truecolor` (`0xRRGGBB`) rendering across Rust core, C fallback, Dart API, raster import, and `FlutterxelView`.
- Added runtime ABI symbols for color mode configuration:
  - `flutterxel_core_set_color_mode`
  - `flutterxel_core_color_mode`
- Added `init(..., color_mode / colorMode)` and runtime helpers:
  - `flutterxel.colorMode`
  - `flutterxel.color_mode`
  - `flutterxel.isTruecolor`
  - `flutterxel.rgb24(...)`
- Updated mode-specific behavior so indexed mode keeps palette semantics while truecolor preserves RGB24 values and treats `pal`, `load_pal`, and `save_pal` as no-ops.

## Upgrade Notes

### For package users

- Existing indexed behavior remains unchanged when `color_mode` is omitted.
- To opt in:
  - `flutterxel.init(..., color_mode: flutterxel.COLOR_MODE_TRUECOLOR)`
  - `flutterxel.init(..., colorMode: flutterxel.COLOR_MODE_TRUECOLOR)`
- If a native backend reports missing symbol errors for `set/get_color_mode`, use artifacts built from this release.

### For maintainers

- Version metadata is aligned at `0.0.12` in:
  - `packages/flutterxel/pubspec.yaml`
  - `packages/flutterxel_tools/pubspec.yaml`
  - `packages/flutterxel/ios/flutterxel.podspec`
  - `packages/flutterxel/src/CMakeLists.txt`
- Native artifact layout remains:
  - Android: `packages/flutterxel/native/android/jniLibs/...`
  - iOS: `packages/flutterxel/ios/Frameworks/FlutterxelCore.xcframework`

# Release v0.0.3

Release date: 2026-03-04

## Published Packages

- `flutterxel` 0.0.3: https://pub.dev/packages/flutterxel
- `flutterxel_tools` 0.0.3: https://pub.dev/packages/flutterxel_tools

Git tag:

- `v0.0.3`

## Highlights

- Added `flutterxel_tools pixel-snap` command for preprocessing external/AI-generated images into palette-quantized pixel-art assets.
- Added command argument validation and forwarding:
  - required: `--input`, `--output`
  - optional: `--colors`, `--overwrite`
- Added shell wrapper at `packages/flutterxel_tools/tool/pixel_snap_image.sh`:
  - validates inputs and overwrite behavior
  - invokes `reference/spritefusion-pixel-snapper` through Cargo
- Added user-facing documentation:
  - package/repo usage docs
  - docs-site guide for pixel-snap asset preprocessing
- Fixed runtime PNG import behavior in `flutterxel`:
  - `Image.fromImage(...)` now decodes binary PNG input
  - `Image.load(...)` supports PNG decode with palette-mapped import

## Upgrade Notes

### For package users

No API/ABI signature changes in `flutterxel`; behavior improved for PNG image import.

### For tooling users

To use `pixel-snap`, ensure Rust/Cargo is installed and run:

```bash
dart run flutterxel_tools:flutterxel_tools pixel-snap --input assets/raw/hero.png --output assets/pixel/hero.png
```

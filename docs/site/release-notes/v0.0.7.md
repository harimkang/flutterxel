# Release v0.0.7

Release date: 2026-03-04

## Published Packages

- `flutterxel` 0.0.7: https://pub.dev/packages/flutterxel
- `flutterxel_tools` 0.0.7: https://pub.dev/packages/flutterxel_tools

Git tag:

- `v0.0.7`

## Highlights

- Fixed C fallback image-bank sizing regression by restoring `DEFAULT_IMAGE_BANK_SIZE` to `256`.
- Added regression tests for:
  - sprite-sheet row addressing using high y coordinates in image banks
  - frame-width `blt` sampling beyond `16px`
- Bundled prebuilt Rust native artifacts in package `native/` layout:
  - Android `jniLibs`: `arm64-v8a`, `armeabi-v7a`, `x86_64`
  - iOS: `FlutterxelCore.xcframework`, `libflutterxel_core.a`

## Upgrade Notes

### For package users

- Existing public runtime API is unchanged.
- Mobile consumers can use the release package with bundled Rust native artifacts.
- Sprite-sheet rendering in C fallback mode now correctly supports larger image-bank coordinate ranges.

### For maintainers

- `0.0.7` release package was produced after local Android+iOS Rust artifact build and bundle verification.
- Release validation path remains tag-triggered CI dry-run (`publish_dry_run.yml`) followed by manual pub.dev publish.
